- The "Full server" backup option was not tested as part of this configuration.
- The "VSS copy backup" option backs up all files and preserves application log files, unlike "VSS full backup," which clears them. See [Environment Configuration.md](Environment%20Configuration.md) for how this choice affects your backup schedule.
- Windows Server Backup offers three storage destination types: a hard disk dedicated for backups, a volume, and a shared network folder.
  - **The dedicated hard disk type is NOT supported by EXPRESSCLUSTER. Do not use it — a Blue Screen will occur when it is being configured.**
- If you need to recover files from the backup on the standby server, only the latest backup snapshot will be available:
  - Once the backup storage (mirrored by EXPRESSCLUSTER) is failed over to the standby server, VSS functionality is lost, meaning only the LATEST snapshot is available to restore from. If you try to restore files from the remote/standby server, previous snapshots will not show up as available options.
- **Add the script resource** (using the script files from `wsbackup_script.zip` included in this repository — see [Add Script Resource.md](Add%20Script%20Resource.md) for setup) to the failover group when configuring the EXPRESSCLUSTER cluster. This script dismounts and mounts the mirror disk volume every time the group is started on a server. If the mirror disk is manually failed over to the secondary server and then failed back, the snapshots created before the failover will be preserved, allowing data recovery from previous snapshots. Without the script, failing the mirror disk over and back means only the latest snapshot can be used for data recovery.
- **Workaround if the script resource isn't in place:** reboot the primary server, preferably while the mirror disk is still failed over to the standby server. This makes previous snapshots available for recovery once the mirror disk is failed back to the primary server. Be cautious: if a failover/failback occurs without the script in place, and another backup is made while the drive is controlled by the primary server before the reboot happens, previous snapshots will be lost.
